# Naming Conventions

This document defines the naming standards for all dbt models, columns, and tests in this project. All new models and columns must follow these conventions. Existing models should be updated to comply as capacity allows.

---

## Models

| Layer | Format | Example |
|---|---|---|
| Bronze | `brz_<source>_<entity>` | `brz_raw_provider` |
| Silver | `slv_<domain>_<entity>` | `slv_provider_registration` |
| Gold | `gld_<business_concept>` | `gld_registration_summary` |

- Lowercase snake_case throughout
- Domain prefix in silver aligns to subject areas (provider, case, worker)
- Gold names reflect the business concept, not the underlying table structure

---

## Columns

### Keys

| Type | Convention | Example |
|---|---|---|
| Surrogate primary key | `<entity>_key` | `provider_key` |
| Natural / source key | `<entity>_id` | `provider_id` |
| Foreign key | `<referenced_entity>_key` | `provider_key` |

- Surrogate keys are MD5 hashes generated by the `generate_surrogate_key` macro
- Natural keys are preserved from source with `_id` suffix
- Foreign keys use the same name as the primary key they reference

### Dates and Timestamps

| Type | Convention | Example |
|---|---|---|
| Date (no time component) | `<event>_date` | `submitted_date`, `approved_date` |
| Timestamp (with time) | `<event>_at` | `created_at`, `updated_at` |
| Audit loaded date | `_loaded_date` | `_loaded_date` |

### Flags and Booleans

| Convention | Example |
|---|---|
| `is_<condition>` | `is_approved`, `is_high_risk`, `is_duplicate` |

- Always boolean (TRUE/FALSE), never 0/1 or Y/N
- Prefixed with `is_` without exception

### Status Fields

| Convention | Example |
|---|---|
| `<entity>_status` | `registration_status`, `provider_status` |

- Values are title case strings: `Approved`, `Rejected`, `Under Review`, `Submitted`
- Standardised in bronze, never left as raw source values

### Source Reference Columns (Bronze only)

| Convention | Example |
|---|---|
| `_source` | `_source` |

- A literal string identifying the source system
- Added in bronze, carried through to silver

---

## Tests

Every model must have the following tests defined in yml:

| Test | Applied to |
|---|---|
| `unique` | All primary keys |
| `not_null` | All primary keys and mandatory foreign keys |
| `accepted_values` | All status and categorical fields |
| `relationships` | All foreign keys |

---

## Tags

| Tag | Usage |
|---|---|
| `pii` | Columns containing personally identifiable information |
| `bronze` / `silver` / `gold` | Layer tag on every model |
| `domain:<name>` | Subject area tag, e.g. `domain:provider` |

---

## What to Avoid

- Never use abbreviations unless universally understood (e.g. `id`, `dt` for date is **not** acceptable — use `_date`)
- Never name a column just `id`, `key`, `name`, `date` — always prefix with the entity
- Never use `flag` as a suffix — use `is_`
- Never mix cases within a layer (no `providerID` or `ProviderName`)
